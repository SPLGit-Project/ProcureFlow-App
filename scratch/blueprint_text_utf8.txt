1.1  Replication
 Topology
Every 
BundleConnect
 site runs a pair of MySQL 5.7 instances on a single Windows host. The master instance on port 3306 is the operational write-primary; the replica on port 3307 is configured as 
read_only
=ON and streams changes from the master via MySQL binary-log replication. SPL's monitoring and reporting tools connect exclusively to the replica port 3307.
The 
Replicate_Rewrite_DB
 parameter remaps the master database name to a different schema name on the replica, allowing the same host to carry both schemas simultaneously without collision.

12.7  
rfidcust

There are two customer master tables: 
rfidcust
 and debtors. They are structurally similar (both have DEB_KEY, TRADING_NAME, address fields) but serve different roles:
rfidcust
 vs debtors
debtors
Fully populated at all 7 sites (395â€“1,358 rows). This is the PRIMARY customer master. All billing, invoicing, and RFID scanning uses DEB_KEY from debtors.
rfidcust
Populated ONLY at Albany (15 rows). At all other sites 
rfidcust
=0 rows. This table appears to be a customer-facing restricted view or 
BundleConnect
 customer portal user table, not the operational debtor master.
Key difference
rfidcust
 uses a 6-char ALPHA code; debtors 
uses

rfidcust
 has AUTO_DESPATCH, 
login_name
, 
ms_pw

ALWAYS query debtors for customer lookups. Never rely on 
rfidcust
 being populated.
